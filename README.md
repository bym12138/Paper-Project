# 生成发行说明的研究

## 项目简介
本项目旨在探索如何高效生成发行说明（Release Notes），以满足现代软件开发中快速迭代和版本管理的需求。我们研究了现有方法和工具，提出了一些创新性改进，并实现了一些原型。

---

## 研究背景
发行说明是软件开发生命周期中至关重要的组成部分。它帮助开发者、用户和利益相关者了解新版本的变化。然而，手动编写发行说明费时且易出错。本项目的目标是通过自动化生成发行说明，提高开发效率，同时保证内容的准确性和可读性。

---

## 研究目标
1. 对软件的提交信息进行分类。
2. 对类别和句子进行噪音处理。
3. 生成高质量的软件发行说明。
4. 探讨自然语言处理（NLP）技术在生成高质量发行说明中的应用。

---

## 主要研究点
1. **输入数据来源**：
   - 代码提交记录（Git Commit Logs）。
   - 拉取请求（Pull Requests）。
   - 问题跟踪系统（Issue Trackers，如 GitHub Issues、Jira）。

2. **信息筛选与分类**：
   - 如何筛选出对用户或开发者重要的信息。
   - 按类别（Bug 修复、功能增强、性能优化等）组织内容。

3. **自然语言生成（NLG）**：
   - 将技术性描述转化为用户友好的语言。
   - 自动化生成标题和摘要。

4. **模板化与自定义**：
   - 支持用户定义模板，以适配不同的风格和需求。
   - 提供多种语言支持。

5. **集成与扩展性**：
   - 集成现有 CI/CD 流程（如 GitHub Actions、Jenkins）。
   - 支持多种版本控制系统（如 Git、SVN）。

---

## 方法与工具
- **自然语言处理**：
  - 使用 NLP 技术提取关键信息。
  - 应用模型：BERT、GPT 等。

- **数据处理**：
  - 利用 Git 提交历史进行差异分析。
  - 将问题和拉取请求的描述映射到发行说明。

- **自动化工具**：
  - 比较的工具：`release-drafter`、`semantic-release`。
  - 我们构建了一个定制工具，支持更细粒度的控制。
